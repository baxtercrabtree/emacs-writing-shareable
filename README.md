# Shareable Emacs Config

This is a side-by-side Emacs configuration designed to replace a personal `~/.emacs.d` gradually without touching the existing setup. It is organized as a small, shareable starter config with optional modules and local overrides.

## Goals

- keep your current `/Users/bcrabtree/.emacs.d` intact while you test the replacement
- bootstrap packages instead of committing `elpa/`
- separate reusable config from personal machine state
- keep the Org writing workflow available as an optional module
- exclude `org-roam`, `emacsql`, `emacsql-sqlite`, and any database-backed notes setup

## Layout

- `init.el`: thin entrypoint that loads the rest of the config in order
- `early-init.el`: startup defaults that must run before normal init
- `lisp/core/`: shared layers for package bootstrap, settings, defaults, UI, editing, and module loading
- `lisp/modules/`: optional features such as theme support and Org writing
- `local/init.el`: untracked personal overrides for paths, fonts, and enabled modules
- `local/custom.el`: untracked Custom UI state generated by Emacs
- `var/`: untracked runtime state for backups and autosaves

## How It Fits Together

The startup model is intentionally simple:

1. `init.el` establishes the config root and updates `load-path`.
2. `lisp/core/core-bootstrap.el` ensures package management and `use-package` are ready.
3. `lisp/core/core-settings.el` defines the user-facing variables and loads `local/init.el` if present.
4. The remaining `lisp/core/` files apply shared behavior for UI, editing, and defaults.
5. `lisp/core/core-modules.el` loads only the optional modules named in `crabs-enabled-modules`.

This keeps the shared repo readable while still allowing machine-specific customization in `local/init.el`.

## First Run

1. Copy `local/init.example.el` to `local/init.el`.
2. Edit `local/init.el` to set your paths, fonts, and optional settings.
3. Launch Emacs with this config directly:

```sh
/Applications/Emacs.app/Contents/MacOS/Emacs --init-directory=/Users/bcrabtree/.emacs.d-shareable
```

For a one-off validation without opening the GUI:

```sh
/Applications/Emacs.app/Contents/MacOS/Emacs --batch --init-directory=/Users/bcrabtree/.emacs.d-shareable --eval '(message "shareable config loaded")'
```

## Optional Modules

Modules are controlled with `crabs-enabled-modules` in `local/init.el`.

- `theme-modus`: Modus themes plus optional circadian switching
- `org-writing`: writing-focused Org behavior, journal support, `olivetti`, and `wc-mode`

The default shared config enables only `theme-modus`. Add `org-writing` when you want the writing workflow.

## Local Settings

Configure these in `local/init.el` as needed:

- `crabs-enabled-modules`
- `crabs-org-directory`
- `crabs-org-journal-directory`
- `crabs-default-fixed-font`
- `crabs-default-variable-font`
- `crabs-ispell-program`
- `crabs-enable-circadian`
- `crabs-calendar-latitude`
- `crabs-calendar-longitude`

All of them are optional. Startup should succeed even if none are set.

## Migration Notes

- `org-roam` is intentionally removed from this config.
- No Org database packages are installed or configured.
- Personal state such as bookmarks, package downloads, and autosave metadata is not tracked.
- Use your current `.emacs.d` as the baseline while comparing behavior and deciding what to keep.

## Troubleshooting

- If a font is missing, Emacs falls back to its default font.
- If `ispell`, `aspell`, or `hunspell` is missing, spell-check setup is skipped.
- If the Org directories are unset or missing, the `org-writing` module loads without agenda or journal paths.
